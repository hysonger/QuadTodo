## Context

当前应用是一个单页面的四象限待办应用，没有使用路由。顶栏左侧目前显示应用图标和固定标题"四象限待办"，右侧仅有深色模式切换按钮。

**现状**:
- `App.vue` 中的 header 组件包含左侧的图标+标题和右侧的深色模式切换
- `settingsStore.ts` 已存在，使用 LocalStorage 存储设置
- 无路由系统，使用简单的条件渲染

**约束**:
- 不引入 Vue Router，保持简单
- 使用现有的 Pinia + LocalStorage 模式
- 样式需与现有深色模式兼容

## Goals / Non-Goals

**Goals:**
- 移除顶栏左侧的图标和标题"四象限待办"
- 替换为用户可自定义的文字区域，默认显示"日拱一卒"
- 文字使用较小的字号和较浅的颜色，视觉上弱化
- 在顶栏右侧添加设置图标按钮
- 实现设置页面，允许用户配置自定义文字并保存
- 使用 LocalStorage 持久化自定义文字

**Non-Goals:**
- 不实现完整的路由系统（使用条件渲染或简单切换）
- 不添加其他设置项（仅自定义文字）
- 不修改四象限的核心功能

## Decisions

### D1: 设置页面的实现方式
**选择**: 使用条件渲染 + 全屏覆盖层实现设置页面

**理由**:
- 避免引入 Vue Router，保持应用轻量
- 用户点击设置按钮时显示全屏设置面板，点击返回按钮关闭
- 与现有单页面架构一致

**备选考虑**:
- Dialog 弹窗: 适合简单配置，但全屏页面更适合后续扩展更多设置
- 新页面: 需要路由，引入复杂度

### D2: 自定义文字的状态管理
**选择**: 扩展现有的 `settingsStore.ts`

**理由**:
- 现有 settingsStore 已管理深色模式设置，扩展成本低
- 统一管理用户偏好设置
- LocalStorage 键名可复用 `quad-todo-settings`

**备选考虑**:
- 新建单独的 store: 增加文件复杂度，不必要

### D3: 文字样式设计
**选择**: 使用 `text-sm` (14px) 和 `text-gray-500` / `dark:text-gray-400`

**理由**:
- 略小于默认标题的 `text-base` (16px)
- 灰色系在浅色和深色模式下都有弱化效果
- 与周围元素保持视觉层次

## Risks / Trade-offs

- **无风险**: 实现简单，不影响现有功能
- **潜在交互问题**: 设置页面首次打开时输入框应为空白还是显示当前值？ → **决策**: 显示当前保存的值，让用户可以编辑
